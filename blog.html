Git feature branching is a great way to duplicate a project or part of a project so that you are not working on the original source code. Instead of us duplicating the motoshop project literally on our local computers, with git we can create a branch that duplicates the motoshop and allows us to make any changes from there forward without affecting the original source. Outside of avoiding editing the original source, branching is great for implementing new features to the project. Creating a branch for each of the features of our CPR will let us build and test the features we plan to develop for our cart without breaking what is already completed and functionally working. Through the branching and workflow of git, it gives us a bit of direction and a guide of what we are currently working on and working toward.

It is important to write effective epics and user stories as these are the most essential problems we are trying to solve. These epics are solving what our customers value most in the application. By writing effective user stories, as developers we can identify the most valuable way to deliver the solution. An effective user story should be small, achievable and should drastically cut down on the time to deliver the solution.

Flowcharts are a great way to break down the user and data flow of your project. Essentially it is a chart that shows how a user will get from A to Z and the options that come in between. Psuedo code is great for prototyping as it can break down what needs to occur for something to happen. It can be read by anyone, independent of the programming language, and explains what each line of code should do once development begins.

Week 2 - Research

Part I – React/Redux Research

According to the research material provided, Flux is an architecture and Redux is a library. They both have many similarities such as both use actions and can both be used with React. Some of the biggest differences are that Flux can maintain multiple stores, while Redux only has one. Redux is also by far more popular than Flux in the open source community. Flux also still uses dispatchers, where Redux’s dispatcher is baked in.

Two of the biggest advantages to using Redux with React is that it makes it easier to understand the data flow in your application and it also centralizes the state. It works well with using state displayed in different locations. So instead of having to lift state up, it is located in the Redux store which is accessible by all components. This is an advantage of using Redux in our cart revamp. Since we need a lot of the data to be usable in multiple components, without the headache of lifting state up with using only React, Redux comes to the rescue creating aa central store for all the state pertaining to our app.
